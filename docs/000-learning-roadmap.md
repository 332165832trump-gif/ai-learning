# 000 - 学习路线总纲

## 为什么我要学编程

我一直以为编程就是"敲代码比谁快"。但在 AI 时代，这条旧逻辑崩塌了——AI 敲代码比任何人都快。

**真正值钱的不是"比 AI 敲代码快"，而是：**

> 懂业务 + 会拆问题 + 会指挥 AI + 会验证结果 + 能把流程自动化

我可以不会自己写 1000 行代码，但我必须能：看懂代码在干什么、判断 AI 写的对不对、把大问题拆成小步骤、把重复劳动自动化。

## 三条主线

### 主线 1：数据获取

**目标**：从互联网和 API 获取有价值的数据。

**核心能力**：`requests`、`BeautifulSoup`、`pandas`、正则表达式、JSON、API 调用、反爬基础认知。

**合法方向**：财经新闻抓取、上市公司公告抓取、基金净值抓取、宏观经济指标抓取、币价行情抓取。

### 主线 2：自动化执行

**目标**：让程序替我完成重复性操作。

**核心能力**：Playwright、Selenium、文件操作、定时任务、日志记录、异常处理。

**合规项目**：自动整理文件、自动填表、监控网页变化后提醒、自动打开选课页面检测余量。

### 主线 3：金融建模

**目标**：用数据驱动投资决策。

**核心能力**：`pandas`、`numpy`、数据清洗、因子构造、策略回测、风险控制、可视化。

**数据流**：数据源 → 清洗 → 因子 → 回测 → 风控 → 报告。

## 合法合规边界

**我能做的**（合法、合规、对平台友好）：
- 抓取公开网页数据（不绕过反爬、不破解验证码）
- 调用官方 API（有 API Key、遵守频率限制）
- 监控信息变化 → 快速提醒 → 人为确认执行
- 自动填表（非抢购场景）
- 金融数据分析、策略回测、风险监控

**我不能做的**（破坏公平规则、违反平台条款）：
- 绕过验证码、绕过排队机制
- 批量抢票、违规抢课
- 自动下单抢购
- 规避平台限制、伪造身份

> 真正高级的路线不是"写抢票外挂"，而是学会：**监控信息变化 → 快速提醒 → 合规辅助决策 → 人来确认执行**。

## 四阶段学习路线

### 第一阶段：Python 自动化基础（✅ 已完成）

**掌握的技能**：文件操作、`os` 模块、`for` 循环、条件判断、路径处理、异常处理基础。

**完成的项目**：文件自动分类器。

### 第二阶段：网页数据采集（🔜 进行中）

**待掌握**：`requests`、`BeautifulSoup`、`pandas`、正则表达式、API 请求、JSON 处理。

### 第三阶段：浏览器自动化（🔜 待开始）

**待掌握**：Playwright、Selenium、表单填写、网页状态监控、截图、自动登录。

### 第四阶段：金融数据能力（🔜 待开始）

**待掌握**：数据清洗、因子构造、策略回测、风险控制、报告生成。

## 为什么这些能力对量化有帮助

量化交易的底层逻辑就是：**获取数据 → 清洗处理 → 构造信号 → 回测验证 → 风险控制 → 执行决策**。

这和文件分类器项目一模一样：**读取数据 → 遍历判断 → 执行动作**。

你今天学的是文件分类器，五年后你可能在用同一套思维做交易信号监控。编程思维是可以迁移的——学会了拆问题，什么领域都能切进去。
