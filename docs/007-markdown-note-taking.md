# 007 - Markdown 记笔记方法

## 1. 为什么用 Markdown

- GitHub 能直接渲染展示，不用下载
- 纯文本文件，任何编辑器都能打开
- 语法简单，5 分钟学会
- 适合记录代码、笔记、文档

## 2. 常用语法速查

| 效果 | 写法 |
|------|------|
| 一级标题 | `# 标题` |
| 二级标题 | `## 标题` |
| 三级标题 | `### 标题` |
| **粗体** | `**粗体**` |
| 行内代码 | `` `代码` `` |
| 代码块 | ` ``` ` 开头和结尾 |
| 无序列表 | `- 条目` |
| 表格 | `\| 列1 \| 列2 \|` |
| 分割线 | `---` |

## 3. 常见错误

**不要在前面加反斜杠**：

```markdown
# 正确 ✅
\# 错误 ❌（前面多了 \，GitHub 不会正确渲染）
```

代码块也一样：不要在 ` ``` ` 前面加 `\`。

## 4. 我的错误笔记模板

每个错误笔记用这个格式：

```markdown
### 编号 - 错误标题

**我遇到的问题**：（描述当时看到的现象）

**报错信息**：（粘贴完整的错误信息）

**错误原因**：（用初学者能懂的话解释为什么出错）

**正确写法**：（给出正确的代码）

**我的理解**：（用自己的话复述一遍，加深记忆）

**下次如何避免**：（总结一条简单规则）
```

## 5. 为什么记错误笔记比记正确答案更重要

1. **错误是最佳学习材料**：你踩过的坑，三个月后回头看，比看十遍教材记忆更深刻
2. **形成错误模式库**：积累多了你会发现，同样的错误类型反复出现
3. **Debug 能力来自经验**：下次看到类似报错，你能立刻定位问题

**我最重要的学习习惯**：每次报错 → 理解原因 → 记入笔记 → 告诉自己"下次如何避免"。
