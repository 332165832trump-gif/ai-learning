# 008 - ChatGPT + OpenCode + Codex 协作工作流

## 1. 我的工具分工

| 工具 | 角色 | 做什么 |
|------|------|--------|
| **ChatGPT** | 教练 | 规划学习路线、解释概念、纠正错误、讲清本质 |
| **OpenCode** | 本地工程师 | 在本地写代码、改文件、运行测试 |
| **Codex** | 云端工程师 | 在 GitHub 创建分支、修改文件、提交 PR |
| **Git** | 记录员 | 记录每次修改的版本，可回溯任意历史 |
| **GitHub** | 档案馆 | 保存和展示完整学习轨迹 |

## 2. 正确学习闭环

```
我提出问题
    ↓
ChatGPT 拆解问题、规划步骤
    ↓
OpenCode 执行（写代码、改文件）
    ↓
我运行代码、理解每一行
    ↓
我写笔记记录理解
    ↓
git status → add → commit → push
    ↓
GitHub 保存学习轨迹
```

**关键**：不是"AI 写完了事"，而是**我理解 → 我记录 → 我存档**。只跑代码不理解的，等于没学。

## 3. 给 AI 的好提示词结构

```
目标 + 当前状态 + 限制条件 + 输出格式 + 不要做什么 + 完成后汇报
```

**示例**：
> 目标：把文件分类器项目加上 os.rename 移动功能
> 当前状态：已经有了分类判断和打印，代码见附件
> 限制：每次只改一个分支、每行写解释、不要一次给完整代码
> 输出：先解释原理 → 再写代码 → 再解释结果

## 4. OpenCode 提示词模板

```
每次只做一步
每行代码解释
不要一次写完
先解释 → 再写代码 → 再运行 → 再解释结果
修改后告诉我改了什么、建议什么 commit message
```

## 5. Codex 提示词模板

```
只改 notes.md（指定文件）
不改代码文件
保留原有内容
在末尾追加
不要用 \# 给标题加反斜杠
完成后告诉我：修改了什么、建议的 commit message
```

## 6. 我真正要培养的能力

不是"比 AI 写代码快"，而是：

| 能力 | 说明 |
|------|------|
| 会定义问题 | 把模糊需求变成清晰目标 |
| 会拆解问题 | 把大任务拆成可执行的小步骤 |
| 会判断 AI 对不对 | 审查代码、找出逻辑漏洞 |
| 会修改 AI 的代码 | 不是照单全收，而是能改出自己需要的 |
| 会把项目沉淀到 GitHub | 学习轨迹可追踪、可复盘、可展示 |
